## 1. Context

v0.5.0 is a coordinated breaking-change release. The bulk of the backward
compatibility shim removal is already complete (¬ß2‚Äì¬ß17 of the
[shim removal plan](v050-backward-compatibility-shim-removal-plan.md)).
Schema migrations 000030 and 000031 have landed. Config restructure Phases 1
and 2 are complete.

This plan consolidates the **remaining** technical debt items across the
codebase, classifies each as remove-now vs retain/defer, and provides a
single checklist for the final cleanup sprint before v0.5.0 ships.

### What we are keeping

| Item | Reason | Removal target |
| ---- | ------ | -------------- |

## 6. Decision Notes

1. **VRLOG speed-key fallback is explicitly retained.** The migration window
   for pre-#352 `.vrlog` files has not closed. Removing it now would silently
   lose speed data on replay. Deferred to v0.5.2 per shim removal ¬ß18.

2. **Deploy tool is explicitly retained.** Removal is gated on four conditions
   in the retirement gate (image pipeline operational, packaging confirmed,
   migration period elapsed, zero active users). None are met.

3. **Config migration tool is retained.** Users migrating from v0.4.x need
   `make config-migrate` to convert their flat tuning JSON to v2 format.
   The tool will be removed when the migration window closes.

4. **Type aliases are architectural, not compat.** The `internal/lidar/`
   package aliases provide a stable package boundary. They are not legacy
   shims and will not be removed.

5. **Sprint scope is deliberately conservative.** Only items with clear scope,
   limited blast radius, and existing test coverage are included. Structural
   debt (god files, context propagation, race conditions) is deferred to
   v0.5.1 where it can receive proper attention.
